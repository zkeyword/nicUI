JS规范 - 代码格式

1)、变量声明必须使用var
2)、使用分号结尾

js语法是允许结束语句不用分号，在代码没合并前运行是正常的，在代码合并后就会出错，使用分号结尾可以避免类似以下的错误：

/* 合并前，正常 */
var x = 1
var x =2
/* 合并后，出错 */
var x = 1var x =2

3)、总是使用大括号

每个条件都放在同一行，这样不容易出错也比较容易阅读，例如：

/* 没有大括号容易出错 */
if( isNaN(x) ) alert('x');

/* 容易阅读维护方便 */
if( isNaN(x) ){
	alert('x');
}

4)、注释规范，使用JSDoc，语法基本遵守JSDoc注释规范。
5)、使用三目运算符(?:)取代if判断语句，充分利用&&与||运算符来缩短代码

/* if语句 */
if( x ){
	alert('x');
}else{
	alert('y');
}

/* 三目运算符 */
x?alert('x'):alerty('y');

/* 更方便 */
var y = x?x:y;

6)、“，”之后要留空格，例如：

function test( x, y, z ){

}

7)、带参函数的“(”和“)”前后应当加空格，例如：

alert( x );

8)、左花括号{ 放于关键字或方法名的同一行末尾，通常右花括号}独立成行，不与任何语句并列一行。

js语法是允许结束语句不用分号，如果没有分号，js默认在换行的时候结束语句，典型事例：

/* 出错 */
return 
{
	'x': 1
}

/* 正确 */
return {
	'x': 1
}

9)、在每个函数定义结束之后都要加空行，例如：

function good(){ }
…//空行

10)、不同的功能实现间应当空行，便于代码的浏览，例如：

function func(){
	/*获取和设置相关的属性功能*/
	var id = $("#id");
	id.style…
	id.value… ..
	//此处应当空一行 
	/*其它小功能区*/
}

11)、缩进应该是每行一个Tab(4个空格)，不要在代码中使用Tab字符。
12)、如果代码行中的运算符比较多，用括号确定表达式的操作顺序，避免 使用默认的优先级。

if ((a | b) && (a & c))

13)、变量命名，建议以类型前缀+有意义的单词组成，用驼峰式命名法增加变量和函式的可读性。

s：表示字符串，例如：sName，sHtml；
n：表示数字，例如：nPage，nTotal；
b：表示逻辑布尔，例如：bChecked，bHasLogin；
a：表示数组，例如：aList，aGroup；
r：表示正则表达式，例如：rDomain，rEmail；
f：表示函数，例如：fGetHtml，fInit；
o：表示以上未涉及到的其他对象，例如：oButton，oDate；
g：表示全局变量，例如：gUserName，gLoginTime；
